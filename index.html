# projectlibre # juhend # kalender

Kohandatud kalender
ProjectLibre's

Kuidas luua ja rakendada kohandatud tööajakalendrit — samm-sammult.

6 sammu ~5 min ProjectLibre 1.x
// sissejuhatus

ProjectLibre on tasuta ja avatud lähtekoodiga projektihaldustarkvara — alternatiiv MS Projectile. Vaikimisi kasutab ProjectLibre standardkalendrit — tööaeg kell 08:00–17:00, tööpäevad esmaspäevast reedeni. Kui sinu projekt toimib muudel kellaaegadel (nt pärastlõunane vahetus, öövahetus) või vajab kohandatud puhkepäevi, tuleb luua kohandatud baasikaender ja rakendada see projektile.

01

Ava ProjectLibre ja mine „Define Calendars"

Ava oma projektifail ProjectLibre's. Mine ülaribalt Project → Define Calendars. Avaneb kalenderihalduse dialoog, kus on näha vaikimisi kalender Standard.

// samm_01 — Project → Define Calendars
📅
💡
Vihje: ProjectLibre's asub kalenderihaldus menüüs Project, mitte eraldi tööriistaribal. Veendu, et projekt on avatud enne menüüsse minemist.
02

Loo uus kalender

Klõpsa New (Uus). Sisesta kalendrile selge nimi, nt Pärastlõunane vahetus. Vali aluseks Standard-kalender, et säiliksid põhiseaded (tööpäevad E–R). Klõpsa OK.

// samm_02 — uue kalendri loomine
➕
03

Määra tööajad

Vali loodud kalender loendist. Klõpsa vahekaardil Work Weeks (Töönädalad). Vali [Default] rida ja klõpsa Details. Märgi kõik soovitud tööpäevad (nt E–R) ja sisesta täpne tööaeg:

  • From: 14:00
  • To: 18:00

Klõpsa OK, seejärel rakenda muudatused.

// samm_03 — tööaegade määramine
04

Rakenda kalender projektile

Sulge kalendrihalduse dialoog. Mine File → Project Information (või Project → Project Properties). Otsi välja Calendar rippmenüü ja vali oma loodud kalender. Klõpsa OK.

// samm_04 — kalender projekti külge
🔗
05

Kontrolli Gantt-diagrammi

Pärast kalendri rakendamist uuendab ProjectLibre automaatselt ülesannete ajastuse. Ava Gantt-vaade (View → Gantt Chart) ja kontrolli, et ülesanded algavad uue kalendi järgi (nt 14:00 asemel mitte 08:00).

// samm_05 — Gantt kontrollimine
📊
💡
Ikka ei uuene? Proovi sulgeda ja uuesti avada projektifail. Kontrolli ka, et ülesannetel ei ole kõva alguskuupäeva (Constraint: Start No Earlier Than), mis võib tühistada kalendri mõju.
