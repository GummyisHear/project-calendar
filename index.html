How to Create and Apply a Custom Calendar

Change working hours for your entire project — step by step.

By default, MS Project schedules tasks using the Standard calendar — working hours from 8:00 to 17:00. If your project runs on different hours (an afternoon shift, a night shift, or a weekend schedule), you need to create a custom base calendar and apply it to the project. This guide walks through the full process.

1

Open Change Working Time

Go to the Project tab in the ribbon and click Change Working Time.

📷 Screenshot: Project tab → Change Working Time button highlighted
2

Create a new calendar

In the dialog that opens, click Create New Calendar. Give it a clear name (e.g. Afternoon Shift) and choose Make a copy of Standard calendar so the base settings carry over.

📷 Screenshot: "Create New Calendar" dialog with name filled in
3

Set your working hours

Still in the Change Working Time dialog, click the Work Weeks tab → select the default row → click Details. Select all working days (Monday through Friday), choose Set day(s) to these specific working times, and enter your hours — for example 14:00 to 18:00. Click OK.

📷 Screenshot: Work Weeks → Details dialog with custom hours entered
4

Apply the calendar to the project

Go to Project tab → Project Information. In the Calendar dropdown, select your new calendar. Click OK.

📷 Screenshot: Project Information dialog with custom calendar selected in dropdown
5

Set the default start time

Go to File → Options → Schedule. Change Default start time to match your calendar (e.g. 14:00). This must match the calendar or tasks may not align correctly.

📷 Screenshot: File → Options → Schedule tab with Default start time set to 14:00
6

Switch all tasks to Auto Scheduled

This is the most common reason the calendar seems to have no effect. Select all tasks (Ctrl + A), then go to the Task tab and click Auto Schedule. Check the bottom status bar — it should say "New Tasks: Auto Scheduled".

📷 Screenshot: Task tab → Auto Schedule button, and status bar showing "Auto Scheduled"
⚠ Common pitfall: If the status bar says "Manually Scheduled", the tasks ignore all calendar and constraint logic entirely. Auto Schedule must be active for the calendar to take effect.
7

Recalculate the schedule

Press F9 or go to Project → Calculate Project. All tasks will be rescheduled according to the new calendar. Your tasks that previously started at 08:00 will now start at 14:00.

📷 Screenshot: Project tab → Calculate Project button, and Gantt chart showing updated start times
💡 Still not updating? Go to Project Information, change the project start date to any other date, click OK, then change it back to the correct date and click OK again. This forces a full recalculation from scratch.

Quick checklist

  • Created a new base calendar with correct working hours
  • Applied the calendar in Project Information
  • Set Default start time to match in File → Options → Schedule
  • Switched all tasks to Auto Scheduled
  • Pressed F9 to recalculate
