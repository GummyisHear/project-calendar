# projectlibre # diagrammid # gantt

Diagrammid
ProjectLibre's

Gantt-diagramm, ressursside diagramm ja võrgudiagramm — ülevaade ja samm-sammult juhend.

3 diagrammitüüpi ~8 min ProjectLibre 1.x
// sissejuhatus

ProjectLibre pakub mitmeid visuaalseid tööriistu projekti planeerimiseks ja jälgimiseks. Peamised diagrammitüübid on Gantt-diagramm (ajakava visualiseerimine), ressursside diagramm (töökoormus inimeste ja vahendite kaupa) ning võrgudiagramm ehk PERT-diagramm (ülesannete sõltuvuste kaardistamine). Kõik vaated on kättesaadavad menüüst View.

// diagrammitüüp_01

Gantt-diagramm

Gantt-diagramm on projektijuhtimise kõige levinum tööriist. See näitab ülesandeid ajajoonel — kes mida teeb, millal algab ja millal lõpeb, ning millised ülesanded sõltuvad teistest.

01

Ava Gantt-vaade

Mine View → Gantt Chart. Ekraani vasakul poolel on ülesannete tabel, paremal pool graafiline ajaskaala ribadena.

// gantt_01 — View → Gantt Chart
📊 View → Gantt Chart
02

Lisa ülesanded

Klõpsa tühjal real vasakpoolses tabelis ja sisesta ülesande nimi veergu Name. Täida ka Duration (kestus, nt 3d = 3 päeva), Start (algus) ja Finish (lõpp). Eeldusel, et kasutad kohandatud kalendrit, arvestab ProjectLibre automaatselt sinu seadistatud tööaegu.

// gantt_02 — ülesannete sisestamine
✏️ Name · Duration · Start · Finish
03

Määra ülesannete sõltuvused

Ülesannete järjestuse loomiseks klõpsa veeru Predecessors lahtris ja sisesta eelneva ülesande reanumber (nt 2). Gantt-diagrammil ilmub noolte graafiline seos kahe riba vahele. Vaikimisi on sõltuvuse tüüp Finish-to-Start (FS) — ülesanne B algab siis, kui ülesanne A lõpeb.

// gantt_03 — sõltuvuste lisamine
🔗 Predecessors → reanumber
💡
Sõltuvusetüübid: Saad sisestada 2FS (Finish-to-Start), 2SS (Start-to-Start), 2FF (Finish-to-Finish) või 2SF (Start-to-Finish). Kõige levinum on FS.
04

Määra ressursid ülesannetele

Klõpsa ülesandel topelt, et avada Task Information dialoog. Mine vahekaardile Resources. Vali ressursi nimi rippmenüüst (ressursid tuleb esmalt lisada — vt ressursi-juhend allpool). Saad määrata ka ressursi koormuse (Units), nt 50% osaajaga töö jaoks.

// gantt_04 — ressursside määramine
👤 Task Information → Resources
05

Salvesta ja ekspordi Gantt-diagramm

Mine File → Save, et salvestada projektifail (.pod formaat). Diagrammi visuaali eksportimiseks vali File → Print → Print Preview, kus saad ka PDF-i eksportida. Alternatiivina võid teha ekraanitõmmise (screenshot) kujunduse jagamiseks.

⚠
NB! ProjectLibre salvestab faili .pod formaadis, mis ei ole otse ühilduv MS Projecti .mpp failiga. Vajadusel ekspordi .mpx formaadis kokkusobivuse tagamiseks.
// diagrammitüüp_02

Ressursside diagramm

Ressursside diagramm (Resource Graph) näitab graafiliselt, kui palju tööaega iga inimene või vahend projekti vältel kasutab. Aitab tuvastada ülekoormust (punane ribadena) ja ressursside ebatõhusat kasutamist.

01

Lisa ressursid projekti

Mine View → Resource Sheet. Sisesta iga ressursi nimi (nt töötaja nimi), tüüp (Work inimeste jaoks, Material materjalide jaoks) ja maksimaalne saadavus (Max Units, vaikimisi 100%).

// ressurss_01 — View → Resource Sheet
👥 Resource Sheet → nimi · tüüp · Max Units
02

Ava ressursside diagramm

Mine View → Resource Graph. Ekraanil kuvatakse tulpdiagramm, mis näitab valitud ressursi töökoormust ajas. Punased tulbad tähistavad ülekoormust (üle 100%). Saad ressurssi vahetada, klõpsates nimekirjas vasakul allosas.

// ressurss_02 — View → Resource Graph
📈 Resource Graph — töökoormus ajas
💡
Ülekoormuse lahendamine: Kui ressurss on üle koormatud, proovi pikendada ülesande kestust, vähendada ressursi osalusprotsenti (Units) või nihutada ülesannet ajas.
// diagrammitüüp_03

Võrgudiagramm (PERT)

Võrgudiagramm ehk PERT-diagramm kujutab ülesandeid kastidena ja nende vahelisi sõltuvusi nooltena. See on eriti kasulik kriitilise tee (critical path) visualiseerimiseks — millised ülesanded ei tohi hilineda, et projekt lõpeks õigeaegselt.

01

Ava võrgudiagrammi vaade

Mine View → Network Diagram. Ekraanil kuvatakse kõik ülesanded kastidena, mis on ühendatud nooltega vastavalt Predecessors-veeru sõltuvustele. Kriitilisel teel olevad ülesanded on tavaliselt esile tõstetud.

// pert_01 — View → Network Diagram
🕸️ Network Diagram — kastid + nooled
02

Loe võrgudiagrammi

Iga kast sisaldab ülesande nime, numbrit, kestust ning varajast ja hilist algus-/lõppaega. Nooled näitavad sõltuvuse suunda. Punaselt tähistatud kastid kuuluvad kriitilisele teele — nende hilinemisel hilineb ka kogu projekt.

💡
Kriitiline tee: Kriitilisel teel oleval ülesandel on varu (Total Float) = 0 päeva. See tähendab, et ükskõik milline hilinemine selles ülesandes pikendab projekti lõppkuupäeva.
// pert_02 — kriitilise tee lugemine
🔴 Punased kastid = kriitiline tee
